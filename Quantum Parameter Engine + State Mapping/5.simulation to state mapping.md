## What This Phase Actually Fixes

In Task 1 feedback, the reviewer said: *"No formal mapping to deterministic state transitions (needed for Kanishk integration)"*. This was a real gap — I described what happens to the hull but never wrote the actual equations. Kanishk's engine can't use a description. It needs numbers in a specific format.

This phase is that format.

**The key idea:** instead of sending "corrosion depth = 1.24mm", we send "corrosion changed by +0.00034mm this hour". The engine tracks the absolute state itself. We only tell it the **changes (deltas)**.

---

## The State Vector

Every hull zone has a state vector — a snapshot of its current condition:

```
S(zone, t) = {
  corrosion_depth_mm     : float,  // how deep rust has penetrated (mm)
  coating_thickness_mm   : float,  // how much protective coating remains (mm)
  roughness_Ra_um        : float,  // average surface roughness in micrometres
  fouling_coverage_frac  : float,  // fraction of surface covered [0.0 to 1.0]
  fouling_thickness_mm   : float   // average fouling layer thickness (mm)
}
```

Each timestep Δt (default: 1 hour), we compute **ΔS** — how much each variable changes — and send that to the engine.

---

## Equation 1 — Corrosion Delta

This is the core equation. It uses the quantum-derived `k_base` as input:

```
Δcorrosion(z, t) = k_base × f_T(T) × f_S(S) × f_O2(O2) × M_Fe × Δt

Where:
  k_base   = quantum-derived base rate constant  [mol/cm²/s]
  M_Fe     = molar mass of iron = 55.84 g/mol    [converts mol to grams]
  Δt       = timestep in seconds                 [= 3600 for 1-hour step]
  
  f_T(T)   = Arrhenius temperature correction
           = exp(−Eₐ/RT) / exp(−Eₐ/RT_ref)
           = exp(−0.82 × 11605 / T) / exp(−0.82 × 11605 / 293)
           
           Examples:
           T = 22°C (295K): f_T = 1.08  (8% faster than 20°C reference)
           T = 15°C (288K): f_T = 0.91  (9% slower)
           T = 30°C (303K): f_T = 1.29  (29% faster — tropics matter)
           
  f_S(S)   = salinity correction
           = 1 + 0.015 × (S − 35)    [S in parts-per-thousand]
           
           Examples:
           S = 36.2 ppt: f_S = 1.018  (1.8% faster — slight increase)
           S = 30.0 ppt: f_S = 0.925  (Baltic Sea is lower salinity)
           
  f_O2(O2) = dissolved oxygen correction
           = O2 / 8.0                [O2 in mg/L, reference = 8.0 mg/L]
           
           Examples:
           O2 = 7.8 mg/L: f_O2 = 0.975  (2.5% slower)
           O2 = 6.0 mg/L: f_O2 = 0.750  (25% slower — deep water, less O2)
```

The correction factors come from NACE International corrosion standards — not made up. These are empirical fits to decades of real marine data.

## Equation 2 — Coating Degradation Delta

```
Δcoating(z, t) = −k_coat × Δcorrosion(z, t) × coating_thickness(z, t)

Where:
  k_coat               = 0.12     [dimensionless coupling constant]
  coating_thickness    = current coating state variable  [mm]
  
  Applied only when:   coating_thickness > 0.01 mm
  When coating = 0:    corrosion equation uses a multiplier of 1.5 
                       (bare metal corrodes 50% faster — no protection)
```

**Intuition:** Coating degrades faster when corrosion is faster (rust undermines coating from below) and when more coating remains (more surface area for the degradation mechanism). This is a standard first-order degradation model used in marine coatings industry (ISO 12944 standards).

---

## Equation 3 — Surface Roughness Delta

```
Δroughness(z, t) = α_r × Δcorrosion(z, t)  +  β_r × fouling_coverage(z, t) × Δt

Where:
  α_r = 15.3 μm/mm   [roughness increase per mm of corrosion depth]
                     Source: Townsin (2003), "The Ship Hull Fouling Penalty"
  β_r = 2.1 μm/hr    [roughness increase from fouling presence]
                     Source: fitted from barnacle growth data
```

**Why roughness matters:** A smooth hull has roughness ~25 μm (Ra). A corroded hull might reach 200+ μm. This directly increases hydrodynamic drag. The decision engine uses this to estimate fuel penalty:

```
Approximate fuel penalty from roughness:
  ΔR_hull ≈ 0.044 × (Ra_final - Ra_initial)^0.21  [% resistance increase]
  Fuel penalty ≈ ΔR_hull × (required_thrust / engine_efficiency)
  
  Example: Ra goes from 25μm to 200μm
  → ΔR_hull ≈ 5.8% resistance increase
  → ~3.2% additional fuel consumption at 12 knots service speed
```

## Equation 4 — Fouling Delta

```
Δfouling_coverage(z, t) = k_f × (1 − fouling_coverage) × f_vel(v) × Δt

Where:
  k_f          = 0.003 hr⁻¹   [base growth rate in static water]
                 Represents barnacle/algae settlement rate
                 
  f_vel(v)     = exp(−0.8 × v / v_ref)  [velocity inhibits fouling]
  v_ref        = 10 knots
  
  Examples:
  v = 0 knots  (docked):  f_vel = 1.00  (maximum fouling growth)
  v = 10 knots:           f_vel = 0.45  (55% reduction from speed)
  v = 12 knots:           f_vel = 0.38  (62% reduction)
  v = 20 knots:           f_vel = 0.20  (80% reduction)

Fouling coverage is capped at [0.0, 1.0]
```

**Physical reasoning:** Fast-moving water applies shear force that prevents organisms from attaching. Ships docked in harbour foul much faster than ships underway. This is well-documented in marine biology literature.

## Zone-Specific Parameters

Different parts of the hull experience different conditions. The environmental inputs vary by zone:

| Zone | Temp Offset | Salinity | O₂ | Flow Velocity | k_coat | α_r |
|---|---|---|---|---|---|---|
| BOW_PORT | +0°C | ambient | ambient | 1.0× | 0.12 | 15.3 |
| BOW_STBD | +0°C | ambient | ambient | 1.0× | 0.12 | 15.3 |
| MID_KEEL | +0°C | ambient | -10% | 0.3× | 0.10 | 16.1 |
| AFT_STERN | +1°C | +0.5ppt | ambient | 1.5× | 0.14 | 14.8 |
| WATERLINE | +0°C | ±2ppt tidal | +5% | 0.8× | 0.08 | 17.2 |
| BALLAST_TANK | +2°C | variable | -20% | 0.0× | 0.15 | 18.0 |

Notes:
- MID_KEEL: below waterline, less dissolved O₂
- AFT_STERN: slightly warmer (engine heat), higher flow from propeller wash
- WATERLINE: tidal variation in salinity; more oxygen due to surface exposure
- BALLAST_TANK: no flow, warm, stagnant water = fastest fouling and corrosion

## Full State Update — What Gets Sent Each Timestep

```
S(z, t+Δt) = S(z, t) + ΔS(z, t)

ΔS = {
  delta_corrosion_mm    :  Equation 1 result
  delta_coating_mm      :  Equation 2 result  (always negative)
  delta_roughness_um    :  Equation 3 result  (always positive)
  delta_fouling_coverage:  Equation 4 result  (positive, capped at 1.0)
  delta_fouling_thick_mm:  0.01 × Δfouling_coverage × Δt  [empirical linear fit]
}
```

Each of these values also has a confidence interval — see Phase 6 for the full uncertainty model.

## Implementation Notes for Kanishk's Engine

1. **Units are fixed**: mm for depths, μm for roughness, fraction (not %) for fouling coverage. Do not convert — the JSON contract (Phase 7) specifies this.

2. **Coating check before applying equation 2**: if `coating_thickness < 0.01mm`, treat as zero and apply bare-metal multiplier to corrosion.

3. **Fouling coverage bounds**: clamp to [0.0, 1.0] after each update. The equation can produce values slightly above 1.0 if timestep is large.

4. **Timestep flexibility**: all equations are written for arbitrary Δt. Default is 1 hour (3600s). For drydock gap simulations, use Δt = 24h (86400s) to simulate one day at a time.

5. **Temperature in Kelvin**: f_T uses T in Kelvin. Sensor data comes in Celsius — add 273.15 before applying the formula.
